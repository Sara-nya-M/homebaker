Software Requirements Specification (SRS)

Project Name: HomeBaker — Custom Bakery & Order Management System

Version 1.0.0 | Full-Stack Architecture

1. Introduction

HomeBaker is a modern full-stack web application designed to connect local artisan homebakers with custom cake customers. The system enables customers to auto-detect nearby bakeries using geolocation, customize specialty cakes in real time, place automated orders with instant email notifications, and track order progress live.

2. Technology Stack & System Architecture

Layer Technology / Library Purpose
Frontend React.js (Vite), Vanilla CSS3 Dynamic, responsive user interface with Sky Blue design system
Backend Java 17 / Spring Boot 3.3.4 RESTful API endpoints, security, and business logic execution
Database PostgreSQL (Supabase Cloud) Relational data storage for Users, Products, Orders, Reviews, & Chats
Security Spring Security + JWT (JSON Web Token) Role-based authentication & encrypted authorization stateless sessions
External API 1 OpenStreetMap (Nominatim API) Reverse geocoding client location to discover nearby homebakers
External API 2 EmailJS API (@emailjs/browser) Automated real-time email notifications sent to bakers on order placement

3. Hardware & Software Requirements

3.1 Developer & Server Hardware Requirements

  • Processor: Dual-Core 2.0 GHz or higher (Intel Core i3/i5/i7 or AMD Ryzen)
  • RAM: Minimum 4 GB (8 GB recommended for concurrent backend/frontend execution)
  • Storage: 500 MB free hard disk space for project binaries and assets
  • Network: Broadband Internet connection for Supabase Cloud DB & External APIs

3.2 Client (User) Software Requirements

  • Operating System: Windows, macOS, Linux, Android, or iOS
  • Web Browser: Modern HTML5 compliant browser (Google Chrome, Microsoft Edge, Mozilla Firefox, Safari)

3.3 Development Environment Software Specifications

  • Java Development Kit (JDK): JDK 17 or JDK 21+
  • Node.js Environment: Node.js v18.0.0+ & npm v9.0.0+
  • Build Automation: Gradle 8.x / 9.x Wrapper
  • Database Engine: PostgreSQL 14+ / Supabase PostgreSQL

4. Functional Requirements

4.1 Authentication & User Management

  • Dual-role registration system for CUSTOMER and BAKER roles.
  • Encrypted password storage using BCrypt password encoder.
  • JWT-based stateless session management for secure API access.

4.2 Geo-Location & Baker Discovery

  • Auto-detect customer GPS coordinates via browser navigator.geolocation.
  • Reverse geocode coordinates into city names using OpenStreetMap Nominatim API.
  • Filter nearby bakery products based on city match.

4.3 Interactive Custom Cake Builder

  • Dynamic selection of cake shape (Round, Square, Heart), size (0.5 kg to 3 kg), flavor, toppings, and custom message.
  • Live price calculation in Indian Rupees (₹).
  • Automated instant email alert sent to baker via EmailJS upon order submission.

4.4 Order Management & Real-Time Tracking

  • Order status workflow: CONFIRMED ➔ BAKING ➔ READY ➔ DELIVERED.
  • Baker dashboard to accept, update, and manage orders.
  • Customer order tracking timeline dashboard with price breakdowns in ₹.

5. Non-Functional Requirements

  • Security: All passwords hashed with BCrypt; secret JWT keys protecting API endpoints.
  • Performance: REST API response time under 200 ms.
  • Aesthetics: Modern Sky Blue (`#0284c7`), Ice Blue (`#f0f9ff`), and Slate White theme.
  • Availability: 99.9% uptime backed by Supabase Cloud PostgreSQL and PaaS deployment.
